# Stratégie de tests – Threed (Dihya Coding)

## Objectifs
- Couverture maximale : unitaires, intégration, e2e, accessibilité, sécurité, RGPD
- Tests automatisés à chaque commit (CI/CD)

## Types de tests
- Unitaires (Python, JS)
- Intégration (API, plugins)
- End-to-end (scénarios 3D)
- Accessibilité (axe-core, pa11y)
- Sécurité (audit, injection, permissions)

## Bonnes pratiques
- Utiliser les fixtures et helpers du module
- Ajouter des tests pour chaque nouvelle fonctionnalité
- Corriger toute régression immédiatement
