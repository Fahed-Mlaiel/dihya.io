# Checklist professionnelle – Structure du dossier /ai (IA)

1. README.md – Présentation, usages, conventions IA
2. index.js – Point d’entrée JS universel (scripts, API, plugins IA)
3. __init__.py – Initialisation Python (scripts, modules IA)
4. ai.config.js – Configuration IA (paramètres, providers, modèles)
5. main.py – Point d’entrée Python IA (orchestration, CLI, batch)
6. ai_service.py – Service principal IA (API, logique métier)
7. ai_utils.py – Fonctions utilitaires IA (pré/post-traitement, helpers)
8. ai_blueprints/
   └─ webApp.js – Blueprint génération webApp IA
   └─ backendApi.js – Blueprint génération backend API IA
   └─ mobileApp.js – Blueprint génération mobile IA
   └─ plugin.js – Blueprint génération plugin IA
9. scripts/
   └─ generate_ai_project.py – Script génération projet IA
   └─ train_model.py – Script entraînement modèle ML/IA
   └─ infer.py – Script inférence modèle IA
   └─ preprocess.py – Script prétraitement données IA
10. models/
    └─ model.py – Modèle ML/IA principal (exemple PyTorch/TensorFlow)
    └─ model_config.json – Config modèle (hyperparamètres, archi)
    └─ tokenizer.json – Tokenizer ou vocabulaire (NLP)
11. data/

12. providers/

# Règles Lead Dev
- Aucun fichier « optionnel » : tout ce qui est listé doit exister.
- Points d’entrée (index.js, __init__.py, main.py) obligatoires.
- Un README.md et une doc métier (fr/en) toujours présents.
- Un exemple métier réel pour chaque type de fichier (script, modèle, asset, config, provider, plugin, i18n…)
- Structure prête à l’industrialisation, clé en main, sans rien d’inutile ni de manquant.
- Tests ignorés ici (centralisation ultérieure).
