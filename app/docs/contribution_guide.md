# Guide de contribution

Merci de contribuer à ce projet ! Voici les règles à suivre :

- Forkez le repo, créez une branche explicite (`feature/nom`, `fix/bug`, etc.)
- Respectez les conventions de commit (Conventional Commits)
- Documentez toute nouvelle fonctionnalité dans le guide utilisateur et/ou dev
- Ajoutez des exemples ou tests si pertinent
- Ouvrez une Pull Request claire, liez les issues concernées
- Relisez le code des autres (code review bienveillante)
- Suivez le workflow CI/CD (lint, build, tests)

Merci pour votre implication !
