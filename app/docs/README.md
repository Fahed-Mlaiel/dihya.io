# Dossier /app/docs

Ce dossier centralise toute la documentation du projet : guides utilisateurs, guides développeurs, architecture, API, contribution, changelog, templates et assets illustratifs.

## Structure recommandée
- architecture.md : documentation technique et fonctionnelle
- user_guide.md : guide utilisateur final
- dev_guide.md : guide développeur
- contribution_guide.md : guide de contribution
- changelog.md : journal des modifications
- api_reference.md : documentation API
- templates/ : modèles de documentation
- assets/ : diagrammes, captures d’écran, etc.
- index.js, __init__.py : automatisations docs

Chaque fichier doit être maintenu à jour et suivre les conventions du projet.
