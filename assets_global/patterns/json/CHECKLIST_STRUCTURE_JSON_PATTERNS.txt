# Checklist professionnelle – Structure exhaustive du dossier json (patterns)

Ce dossier doit contenir tous les fichiers, templates, manifestes et exemples métiers JSON nécessaires à l’industrialisation, la configuration et l’extension du projet patterns, selon les standards Lead Dev/Architecte.

## 1. Points d’entrée et documentation
- README.md : Présentation, usage, structure, conventions, contacts
- index.js : Point d’entrée JS pour import/export dynamique des templates JSON
- json_manifest.json : Manifestes des templates, conventions, métadonnées
- conventions_json.md : Conventions de nommage, structure, bonnes pratiques JSON
- changelog.md : Historique des évolutions du dossier json
- integration_guide.md : Guide d’intégration multi-stack (React, Node.js, API, etc.)

# Règles
- Aucun fichier optionnel : tout doit exister, même vide ou avec un exemple métier.
- Respecter la structure, la nomenclature et les conventions métiers.
- Ignorer les tests (centralisés ailleurs).
- Adapter la structure si de nouveaux besoins métiers apparaissent.
