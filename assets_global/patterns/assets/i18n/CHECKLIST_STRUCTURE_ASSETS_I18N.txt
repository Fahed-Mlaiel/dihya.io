# Checklist professionnelle i18n assets/patterns

Ce dossier doit contenir :
- Un fichier de checklist de structure (ce fichier)
- Un README.md explicatif
- Un point d’entrée JS (`index.js`) et Python (`__init__.py`)
- Un fichier de manifeste i18n (`i18n_manifest.json`)
- Un fichier de conventions i18n (`i18n_conventions.md`)
- Un fichier de changelog (`CHANGELOG.md`)
- Un fichier de documentation d’intégration (`integration_guide.md`)
- Un fichier de documentation multilingue pour chaque langue/dialecte supporté (ex : `i18n_fr.json`, `i18n_en.json`, `i18n_ar.json`, etc.)

Aucun fichier ne doit être optionnel. Toute langue présente dans l’i18n du projet doit avoir son fichier ici.
